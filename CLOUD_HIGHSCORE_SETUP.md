# V5.1.2 Cloud Highscore Diagnose

Supabase ist im Framework eingebaut und wird über `data/cloud-config.js` geladen.

## Konfigurationsdatei

```js
window.CLOUD_HIGHSCORE_CONFIG = {
  enabled: true,
  provider: "supabase",
  supabaseUrl: "https://...supabase.co",
  anonKey: "sb_publishable_...",
  table: "highscores",
  limit: 20,
  allowAnonymous: true,
  classCode: "default"
};
```

## Erwartete Tabelle `highscores`

```sql
create table if not exists public.highscores (
  id bigint generated by default as identity primary key,
  player_name text,
  class_code text default 'default',
  device_id text,
  mode text,
  title text,
  percent integer,
  score integer,
  total integer,
  duration text,
  rank text,
  app_version text,
  created_at timestamptz default now()
);
```

## Einfache RLS Policies für anonyme App-Nutzung

Nur aktivieren, wenn bewusst gewünscht. Für produktiven Betrieb später sicherer machen.

```sql
alter table public.highscores enable row level security;

create policy "public read highscores"
on public.highscores
for select
to anon
using (true);

create policy "public insert highscores"
on public.highscores
for insert
to anon
with check (true);
```

## Neue Diagnose in V5.1.2

Der Systemstatus zeigt jetzt einzeln:

- Config geladen
- URL vorhanden
- Key vorhanden
- Provider
- Tabelle
- Klasse
- Lesetest
- HTTP Status oder Supabase Fehlermeldung

Wenn Config, URL und Key auf „Ja“ stehen, aber der Lesetest fehlschlägt, liegt es sehr wahrscheinlich an Tabelle, RLS/Policies, Internet oder CORS.
