# G54.47.14C · Release-Kanäle

Aktive Kanäle: DEV, BETA und PRODUCTION.

- DEV: lokale Entwicklung, keine produktiven Cloud-Schreibvorgänge.
- BETA: echte Testnutzer, getrennte Firebase- und Worker-Ressourcen.
- PRODUCTION: öffentliche Freigabe.

Nicht-produktive Builds zeigen unten rechts eine sichtbare Kanalmarke. Auf öffentlichen Hosts sind lokale Umgebungs-Overrides gesperrt. Domain und Kanal werden durch `js/core/release-channel.js` gegengeprüft.

## Deployment

```bash
cd worker
npm run deploy:beta
npm run deploy:production
```

Vor einem Deployment müssen Secrets, KV-Bindings, Firebase-Projekt und Domain zum gewählten Kanal passen.
