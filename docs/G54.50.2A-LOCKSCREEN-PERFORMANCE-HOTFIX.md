# G54.50.2A · Lockscreen- und Performance-Hotfix

## Behobene Regressionen

1. Die globale UI-Konsistenzregel begrenzte jedes Element mit `role="dialog"` auf 680 px. Dadurch wurde auch die Nova-Vollbildoberfläche begrenzt und der alte Gate-Screen blieb sichtbar.
2. Breite Selektoren wie `[class*="-card"]`, `[class*="-actions"]` und `[class*="-grid"]` wurden auf der gesamten großen DOM-Struktur ausgewertet. Sie wurden durch explizite Komponentenklassen ersetzt.
3. Smoke Runner, Visual Observer, Live Visual QA, State Capture und Regression Diff wurden bei jedem normalen Appstart geladen. Diese Werkzeuge werden nun nur auf localhost oder mit `?qa=1` geladen.
4. Wenn Nova aktiv ist, wird der alte Gate-Screen ausdrücklich ausgeblendet. Die Nova-Oberfläche besitzt wieder uneingeschränkte Vollbildabmessungen.
5. Der Service-Worker-Cache wurde versioniert, damit alte CSS- und Lockscreen-Dateien nicht weiter ausgeliefert werden.

## Unverändert

Login, Registrierung, Demo, Rollen, Berechtigungslogik, Lernmodule und Nutzerdaten wurden nicht verändert.
