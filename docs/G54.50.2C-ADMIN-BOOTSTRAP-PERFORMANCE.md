# G54.50.2C · Admin Bootstrap & Performance Stability

## Admin-Bootstrap
- Einmaliger serverseitiger Bootstrap für den ersten Systemadministrator.
- Der Klartextcode liegt nicht im Frontend oder Functions-Quelltext, sondern nur sein SHA-256-Digest.
- Firestore-Dokument `system/bootstrapAdmin` sperrt den Vorgang nach Erfolg global.
- Existiert bereits ein Adminprofil, wird ein weiterer Bootstrap abgelehnt.
- Bei Teilausfällen werden gesetzte Custom Claims bestmöglich zurückgerollt.
- Normale Teilnahmecodes können weiterhin niemals Adminrechte vergeben.

## Performance
- Dauerhafte Glass-/Backdrop-Blur-Effekte auf zentralen Oberflächen deaktiviert.
- Hover-Transformationen, Filter und dynamische Großschatten entfernt.
- Dekorative Endlosanimationen gestoppt.
- Veraltetes Cinematic-CSS aus Runtime und Service Worker entfernt.
- Fokus-, Status-, Timer-, Aufnahme- und Ladeindikatoren bleiben funktional.

## Deployment
Der Bootstrap-Endpunkt ist eine Firebase Callable Function und muss vor der Verwendung deployt werden:

```bash
firebase deploy --only functions
```

Die reine Veröffentlichung der statischen PWA-Dateien aktiviert den Bootstrap nicht. Nach dem Functions-Deployment muss die Web-App mit demselben Firebase-Projekt verbunden sein.

## Einmalcode
Der Klartextcode wird bewusst nicht in den Projektdateien dokumentiert. Er wird dem Projektinhaber getrennt übergeben. Im Functions-Code befindet sich ausschließlich sein SHA-256-Digest.
