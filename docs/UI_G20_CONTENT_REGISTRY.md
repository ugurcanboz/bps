# UI-G20 Professional Content Registry

Ziel: Deep-Sheet-Inhalte fachlich sauber sortieren, ohne das UI-Fundament zu verändern.

## Grundregel

Das UI bleibt ein Fundament. Die sichtbaren Inhalte werden über eine zentrale Content-Registry gesteuert:

- `BRANCH_CONTENT` legt fest, welche Module in welcher Fachrichtung erscheinen.
- `MODULES` beschreibt Module mit Titel, Beschreibung, Tags und Startmodi.
- Bereiche zeigen keine fachfremden Module mehr automatisch an.

## Sortierung

### IT / FISI
Pflicht: IT-Grundlagen, Netzwerk, Hardware & OS, Logik, Matrizen, Mathe, Englisch, Konzentration, Simulation.
Optional: Python Quest, Deutsch, Allgemeinwissen.

### Kaufmännisch
Pflicht: Kaufmännisches Rechnen, Büro & Verwaltung, Wirtschaft, DIN 5008, Kommunikation, Deutsch, Konzentration, Logik.
Optional: Mathe, Allgemeinwissen, Simulation.
Python Quest ist hier entfernt.

### Sozialpädagogik
Pflicht: Pädagogik, Entwicklung & Bindung, Praxisfälle, Kommunikation & Elternarbeit, Recht / Schutz, Beobachtung & Dokumentation, Deutsch, Konzentration.
Optional: Allgemeinwissen, Simulation.
Python Quest und kaufmännische Spezialmodule sind hier entfernt.

### Allgemeinwissen
Pflicht: Wissen, Deutsch, Englisch, Logik, Mathe, Konzentration, Simulation.
Optional: Matrizen.
Python Quest ist hier entfernt.

## Sichtbare Titel

Technische Begriffe wie „Deep Sheet Menü“ werden im sichtbaren UI nicht mehr als Hauptsprache genutzt. Nutzer sehen stattdessen fachliche Titel wie „IT / FISI Training“, „Kaufmännisches Training“ oder „KI-Coach“.
