# G54.47.15G · API-Sicherheitsaudit

## Ziel
API-Endpunkte gegen manipulierte Requests, Replay, Query-Missbrauch, Request-Smuggling-Indikatoren und Payload-Fuzzing härten.

## Umgesetzt
- strikte Query-Parameter-Allowlist je Endpunkt
- Begrenzung von `hours` am Metrics-Endpunkt auf 1–336
- nur `stream=0|1` am Coach-Endpunkt
- Blockierung von `X-HTTP-Method-Override`
- Erkennung widersprüchlicher `Transfer-Encoding`-/`Content-Length`-Angaben
- optionale Idempotency Keys mit SHA-256-gehashtem KV-Eintrag
- Replay-Erkennung mit HTTP 409
- Produktions-Fail-Closed, wenn Replay-Schutz angefordert ist, aber SECURITY_KV fehlt
- bekannte, redigierte API-Fehlercodes
- unbekannte Endpunkte bleiben 404, falsche Methoden 405
- Größenlimit greift vor JSON-Verarbeitung

## Grenzen
Idempotency Keys sind optional, damit bestehende Clients kompatibel bleiben. Für besonders kritische schreibende Endpunkte sollten sie künftig clientseitig standardmäßig gesendet werden. Eine verteilte Replay-Sperre setzt SECURITY_KV oder einen Durable Object voraus.
